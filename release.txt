Release Procedure
==================

1. Fetch tags and determine the last version:
   git fetch --tags
   git tag --sort=-v:refname | head -5

2. Determine the new version number (e.g., 0.17.0)

3. Review commits since the last release tag:
   git log v0.16.0..HEAD --oneline

4. Update CHANGELOG.txt:
   - Add a new version section at the top
   - Summarize user-facing changes only: new features, UI changes, notable fixes
   - Skip internal/dev-only changes (CI, test coverage, refactors, spec additions, CSS tweaks)
   - Write entries from the user's perspective (what they can see or do differently)

5. Bump version in config/initializers/version.rb:
   - Update APP_VERSION = '0.17.0'

6. Run bin/all_tests to verify everything passes

7. Create the release commit:
   git add CHANGELOG.txt config/initializers/version.rb
   git commit -m "Bump version to 0.17.0"

8. Create and push the git tag:
   git tag v0.17.0
   git push origin main --tags
